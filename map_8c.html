Detailed Description

A simple hash map implementation using separate chaining.

This file implements a dynamically resizing hash map that stores (key, value) pairs. Keys are strings, and any pointer type can be used as the value. Collisions are resolved with linked lists (separate chaining). The table automatically resizes when the load factor exceeds a threshold (LOAD_FACTOR).

Inserts or updates a key-value pair in the hash map.

Inserts a (key, value) pair into the hash map.

Computes the index using hash_function. If the key is already present, its value is updated. Otherwise, a new node is added to the front (or end) of the chain. If the load factor exceeds LOAD_FACTOR, the map is resized.

Parameters

keyNull-terminated string key (will be copied with strdup).

Returns
